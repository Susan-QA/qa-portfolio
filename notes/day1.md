# Tag 1 – QA Grundlagen

## Was ist Software Testing?
- Prozess, um Software auf Qualität zu prüfen
- Ziel: Software funktioniert korrekt und fehlerfrei
- Fehler finden und dokumentieren
- Beitrag zur Nutzerzufriedenheit

## Testarten
- **Funktional**: Prüft, ob Funktionen wie erwartet arbeiten
- **Regression**: Prüft, ob neue Änderungen alte Funktionen nicht kaputt machen
- **Smoke / Sanity**: Kurzer Test, ob die App startet / grundlegende Funktionen laufen
- **Exploratory Testing**: Ohne festes Testskript; Tester erkundet App / Website aktiv

## Testlebenszyklus
1. Testplanung
2. Testdesign / Testfall erstellen
3. Testausführung
4. Fehlerreporting / Bugtracking
5. Abschluss / Review

## Exploratory Testing
- Ziel: Fehler und ungewöhnliches Verhalten entdecken
- Fokus: Nutzerperspektive
- Vorgehensweise:
  1. Aktion durchführen
  2. Ergebnis beobachten
  3. Dokumentieren
  4. Verbesserungsvorschlag
